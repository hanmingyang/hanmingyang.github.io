JavaScript作用域解析

在js中，我们使用var关键字来声明变量。

比如：

var str = 'somting';
console.log(str); //输出somting

在上面用var关键字声明了一个变量str,并初始化了它的值和类型。所以上面的代码可以理解为为两个步骤，例如：

var str;
str = 'somting';

在js中，有一种预解析机制，js在执行的时候，会把所有的声明都提升到当前作用域的顶端。这个怎么理解呢？看看下面的代码：

var a = '123';
var b = 12;
var c = function(){};

在上面，我们声明了3个变量，那么js是怎么来做解析的呢？看下面：

var a,b,c;
a = '123';
b = 12;
c = function(){};

到了这里，相信大家会有点理解了，咱们再来看一段代码：

var i = 'hello';
(function(){

    console.log(i);//这个地方是输出什么值呢？

    var i = 'world';

})();

答案是undefined!

因为上面的代码可以理解为：

var i = 'hello';

(function(){

    var i;

    console.log(i);

    i = 'world';

})();

这么一看是不是就明白了？我们知道用var声明变量，但是没有初始化值的时候，该变量的数据类型是undefined，而undefined类型只有一个值对不对？
下面我们再来理解开始的那句话：

js在执行的时候，会把所有的声明都提升到当前作用域的顶端

什么是作用域呢？别纠结这个问题，大家只需要知道js的作用域只有两种：

1. 全局作用域（Global Scope）
2. 局部作用域

全局作用域，它的定义很简单，如果一个变量或者函数是全局的，那么在任何地方都可以访问它们。在浏览器中，全局作用域其实就是window对象。咱们之前声明的变量，实际上就是在window对象上创建了一个属性。例如：

var i = 10;//window.i = 10

试试看效果是不是一样呢？window对象因为是全局的，所以它的属性，我们可以在任何地方直接访问。

全局变量怎么声明呢？不使用var关键字。

局部作用域呢，说的好理解一点，就是在函数内部声明的变量或者函数。

function test(){

    var i = 10;//试试不用var 全局变量看看效果？

}
test();
console.log(i);//undefined,找不到，因为该变量在函数内部声明的
